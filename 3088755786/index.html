SocketJS

SockJS 网络协议由两部分组成：

  1. 用于识别 SockJS 会话参数的握手
  2. 全双工通信会话

在开始会话之前，SockJS 客户端会向服务器 WS 的 URL+/info来获取服务器的基本信息，包括：

  • websocket: 指定服务器是否需要 WebSocket 传输的属性
  • cookie_needed: 指定服务器是否需要 cookie 支持的属性
  • origins: 允许的来源列表
  • entropy: 随机数生成器的熵源

在 SockJS 开启会话时，客户端会发送请求 /server/session/transport，其中：

  • server: 服务器编号
  • session: 会话编号
  • transport: 传输方式

SockJS 支持的传输方式：

  • WebSocket: 最佳性能，支持全双工通信
  • XHR Streaming: 使用XMLHttpRequest进行流式传输。它使用两个同时的半双工 HTTP 连接（与所有 Comet 技术一样）来模拟全双工
    WebSocket 连接。
  • EventSource: 基于Server-Sent Events的单向通信
  • XHR Long Polling: 使用使用XMLHttpRequest进行长轮询。此传输也使用两个同时的半双工 HTTP 连接来模拟全双工 WebSocket 连接。

客户端在尝试建立连接时，会依次尝试这些传输方式，直到找到一个可用的。

Spring WebSocket

使用 SockJS 时，路径是固定的，无法路径传参，可以使用 query 或者 header 传参。

继承 DefaultHandshakeHandler 重写 determineUser 可以用于识别用户标识。或者通过实现 HandshakeInterceptor 实现 beforeHandshake，用于识别用户标识。

继承 AbstractWebSocketHandler 重写 afterConnectionEstablished 和 afterConnectionClosed，可以用于管理用户会话。

发送消息的时候，通过 redis topic 广播，可以避免服务端多实例，服务端发送消息给客户端时，客户端会话不在当前服务器实例上。
